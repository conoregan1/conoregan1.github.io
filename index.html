Project 1: EAV Health & Performance Monitoring System

An End-to-End Hardware and Software Solution for Fleet Analytics

The Challenge

Fleet managers of commercial Electric-Assisted Vehicles (EAVs) lack the tools to monitor vehicle performance, driver behaviour, and long-term system health. This leads to inefficient operations, unexpected maintenance, and an inability to accurately assess key metrics like payload and vehicle safety in the field.

My Solution: A Full-Stack Approach

To solve this, I designed and developed a complete end-to-end system. This wasn't just a piece of software, but a full stack solution involving a custom-built hardware data logger and a suite of Python-based analysis tools.

  • Embedded C++ firmware with Kalman filtering for accurate real-time orientation tracking and robust data logging to an SD card.
  • A suite of Python-based analysis tools to process the raw data, providing actionable insights for fleet managers.

Key Features & Technical Implementation

The system transforms raw sensor data into high-level insights. Below is a plot showing the successful results of the mass estimation model, which achieved a 6-7% margin of error against real-world test data.

Additional System Capabilities:

Crash Detection & Safety Analysis

The system processes normalised accelerometer data to automatically log and classify impacts as 'small' or 'large' events, providing critical data for safety analysis and preventative maintenance.

GPS Path Visualisation

A tool that ingests the logged GPS data and plots the vehicle's route, colour-coded by time, allowing for trip analysis and driver monitoring.

Robust Hardware & Firmware

The physical data logger includes features like Wi-Fi reconnection logic and IFTTT notifications for critical system events like GPS signal loss or SD card failure, ensuring data integrity.

Project 2: Designing for Engineers at Airborne UK

Improving Workflows with User-Focused Internal Tools

The Challenge

During my year in industry, I inherited two early-stage internal projects: a basic script for quality control and another for environmental monitoring. I was tasked with transforming them into robust, user-friendly tools that would improve the efficiency of the engineering and quality control teams. As the work was proprietary, all visuals shown are from my university report.

Solution 1: The Real-Time Monitoring Dashboard

Problem: Engineers had no centralised way to monitor factory conditions, forcing them to react to issues rather than prevent them.

Solution: I designed and built a real-time dashboard to give engineers an at-a-glance view of 12 critical data types, from temperature and humidity to machine vibration, using a network of RS485 sensors housed in a custom 3D-printed enclosure.

Process Rigour - Hardware Validation:

To ensure the data was trustworthy, I designed and 3D printed custom jigs in SolidWorks to systematically test and validate the roll, pitch, yaw, and acceleration data from the IMU sensors. This process of rigorous validation was crucial for gaining the engineering team's confidence in the tool.

Solution 2: The Automated QC Tool

Problem: The manual quality control process for composite parts was slow and prone to human error, creating a bottleneck in the production line.

Solution: I developed a computer vision tool that allowed QC technicians to automatically verify part geometry against a defined tolerance.

Key Technical Challenge - Solving Image Distortion:

A major obstacle was that the camera lens warped and distorted the images, making accurate measurement impossible. I researched and implemented a camera calibration routine in Python that mathematically undistorted the images, ensuring the geometric analysis was precise and reliable. This was a critical step that unblocked the entire project.

Impact:

These tools significantly improved the workflows for both the engineering and QC teams. I had the opportunity to present the progress and positive impact of these projects at quarterly review meetings to company stakeholders and external funders, which further developed my technical communication skills.
