Project 1: EAV Health & Performance Monitoring System

An End-to-End Hardware and Software Solution for Fleet Analytics

The Challenge

Fleet managers of commercial Electric-Assisted Vehicles (EAVs) lack the tools to monitor vehicle performance, driver behaviour, and long-term system health. This leads to inefficient operations, unexpected maintenance, and an inability to accurately assess key metrics like payload and vehicle safety in the field.

My Solution: A Full-Stack Approach

To solve this, I designed and developed a complete end-to-end system. This wasn't just a piece of software, but a full stack solution involving:

  • A custom-built hardware data logger using an ESP32 microcontroller, GPS, and a 6-axis IMU to capture high-frequency vehicle dynamics.
  • Embedded C++ firmware with Kalman filtering for accurate real-time orientation tracking and robust data logging to an SD card.
  • A suite of Python-based analysis tools to process the raw data, providing actionable insights through a clear user interface.

A high-fidelity mockup of the proposed dashboard, designed to translate complex vehicle data into clear, actionable insights for a fleet manager.

Key Features & Technical Implementation

The system is capable of performing several key analyses, transforming raw sensor data into high-level insights for the end-user. Below is a sample from the mass estimation model.

# This function estimates vehicle mass based on acceleration,
            # motor force, and IMU-derived incline data.
            def estimate_mass(df):
                # Filter for periods of significant movement to ensure data quality
                df_moving = filter_moving_periods(df)
                if df_moving.empty:
                    raise ValueError("No moving periods detected")
                
                # Smooth acceleration data to reduce sensor noise
                window_size = calculate_optimal_window(len(df_moving))
                df_moving['accel_smoothed'] = smooth_data(df_moving['accel_magnitude'], window_size)
                
                # Isolate periods of significant acceleration
                df_accelerating = df_moving[df_moving['accel_smoothed'] > MIN_ACCELERATION].copy()
                if df_accelerating.empty:
                    raise ValueError("No valid acceleration events")
                    
                # Apply physics-based formula to estimate mass at each data point
                df_accelerating['mass_estimate'] = MOTOR_FORCE / (
                    df_accelerating['accel_smoothed'] + 
                    ROLLING_RESISTANCE_COEFFICIENT * GRAVITY + 
                    GRAVITY * np.sin(df_accelerating['incline_angle'])
                )
                
                return df_accelerating

Additional System Capabilities:

  • Crash Detection: The system processes normalised accelerometer data to automatically log and classify impacts as 'small' or 'large' events, providing critical data for safety analysis.
  • GPS Path Visualisation: A tool that ingests the logged GPS data and plots the vehicle's route, colour-coded by time, allowing for trip analysis and driver monitoring.
  • Robust Hardware & Firmware: The physical data logger includes features like Wi-Fi reconnection logic and IFTTT notifications for critical system events like GPS signal loss or SD card failure, ensuring data integrity.

Project 2: Designing for Engineers at Airborne UK

Improving Workflows with User-Focused Internal Tools

The Challenge

During my year in industry, I inherited two early-stage internal projects: a basic script for quality control and another for environmental monitoring. I was tasked with transforming them into robust, user-friendly tools that would improve the efficiency of the engineering and quality control teams. As the work was proprietary, all visuals shown are recreated wireframes and diagrams.

Solution 1: The Real-Time Monitoring Dashboard

Problem: Engineers had no centralised way to monitor factory conditions, forcing them to react to issues rather than prevent them.

Solution: I designed and built a real-time dashboard to give engineers an at-a-glance view of 12 critical data types, from temperature and humidity to machine vibration.

Key Design Feature - Dynamic Threshold Alerts:

The most important user requirement was to be alerted to problems instantly. To achieve this, I implemented a feature where the line on each graph automatically changes from green to red if the data passes a user-set threshold. This simple visual cue transformed the dashboard from a passive data display into an active monitoring tool, allowing engineers to spot issues immediately.

Process Rigour - Hardware Validation:

To ensure the data was trustworthy, I designed and 3D printed custom jigs in SolidWorks to systematically test and validate the roll, pitch, yaw, and acceleration data from the IMU sensors. This process of rigorous validation was crucial for gaining the engineering team's confidence in the tool.

Solution 2: The Automated QC Tool

Problem: The manual quality control process for composite parts was slow and prone to human error, creating a bottleneck in the production line.

Solution: I developed a computer vision tool that allowed QC technicians to automatically verify part geometry against a defined tolerance.

Key Technical Challenge - Solving Image Distortion:

A major obstacle was that the camera lens warped and distorted the images, making accurate measurement impossible. I researched and implemented a camera calibration routine in Python that mathematically undistorted the images, ensuring the geometric analysis was precise and reliable. This was a critical step that unblocked the entire project.

Impact:

These tools significantly improved the workflows for both the engineering and QC teams. I had the opportunity to present the progress and positive impact of these projects at quarterly review meetings to company stakeholders and external funders, which further developed my technical communication skills.
