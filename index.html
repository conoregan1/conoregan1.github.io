Project 1: EAV Health & Performance Monitoring System

Delivering Actionable Fleet Insights with a Custom Full-Stack IoT System

The Challenge & Academic Context

As my major individual research project for my MEng degree, I collaborated with EAV Ltd. to design a high-quality system to monitor the health, performance, and safety of their next-generation e-bike taxi. This work forms the foundation for my final-year group design project, where a team of seven will deliver a complete vehicle design.

The core challenge was to design a cost-effective system to provide fleet managers with actionable insights into vehicle performance, payload, and safety, without requiring expensive new mechanical components.

My Solution: A Full-Stack Approach

I designed and developed a complete end-to-end system, covering the full stack from custom hardware and embedded firmware to a suite of Python-based analysis tools.

  • **Embedded C++ Firmware:** I developed robust firmware for the ESP32 microcontroller that performs on-board sensor fusion using a Kalman filter for accurate orientation tracking and logs all data reliably to an SD card.
  • **Python Analysis Suite:** I wrote a collection of Python scripts to automatically parse, process, and analyse the raw data, providing valuable insights for fleet managers.

Core Analytical Features

The system transforms raw sensor data into high-level insights. The two flagship features of the analysis suite are a mass estimation model and a crash detection algorithm, both designed to give fleet managers crucial data without requiring additional mechanical sensors.

Feature 1: Mass Estimation for Payload Monitoring

This model calculates the total vehicle payload with a 6-7% margin of error, validated against real-world test data. This allows EAV to monitor how the vehicle is being used and optimise battery performance based on load, a critical factor for operational efficiency.

Feature 2: Crash Detection for Safety & Maintenance

By processing normalised accelerometer data, the system automatically logs and classifies impacts as 'small' or 'large' events. This provides critical data for safety analysis and allows fleet managers to proactively flag vehicles that may require inspection after a significant impact, preventing minor issues from becoming major failures.

Behind the Analysis: A Code Snippet

A critical step in the analysis is normalising the accelerometer data to remove the effects of gravity, which requires transforming the gravity vector into the sensor's local reference frame using 3D rotation matrices. This Python function demonstrates how that was achieved.

def normalize_acceleration(df):
    roll = np.radians(df['Roll'].values)
    pitch = np.radians(df['Pitch'].values)
    yaw = np.radians(df['Yaw'].values)
    
    # Global gravity vector
    gravity_global = np.array([0, 0, 9.81])
    
    # Prepare arrays for normalized values
    accel_normalized_vectors = np.zeros((len(roll), 3))

    for i in range(len(roll)):
        # Create rotation matrices for each axis
        R_x = np.array([[1, 0, 0], [0, np.cos(roll[i]), -np.sin(roll[i])], [0, np.sin(roll[i]), np.cos(roll[i])]])
        R_y = np.array([[np.cos(pitch[i]), 0, np.sin(pitch[i])], [0, 1, 0], [-np.sin(pitch[i]), 0, np.cos(pitch[i])]])
        R_z = np.array([[np.cos(yaw[i]), -np.sin(yaw[i]), 0], [np.sin(yaw[i]), np.cos(yaw[i]), 0], [0, 0, 1]])
        
        # Combined rotation matrix
        R = R_z @ R_y @ R_x
        
        # Transform global gravity to sensor's local frame
        gravity_local = R.T @ gravity_global
        
        # Get raw acceleration and subtract local gravity
        accel_local = np.array([df['x-axis'].iloc[i], df['y-axis'].iloc[i], df['z-axis'].iloc[i]])
        accel_normalized = accel_local - gravity_local
        accel_normalized_vectors[i] = accel_normalized

    df[['x-axis_norm', 'y-axis_norm', 'z-axis_norm']] = accel_normalized_vectors
    return df

Project 2: Designing for Engineers at Airborne UK

Boosting Engineering Efficiency with Custom-Built Automation & QC Tools

The Challenge

During my year in industry, I was tasked with transforming two early-stage internal projects into robust, user-friendly tools that would improve the efficiency of the engineering and quality control teams. As this work was proprietary and covered by a confidentiality agreement, I cannot share the source code or final user interfaces. However, the visuals shown below are from my university report and illustrate the key design processes and technical challenges I solved.

Solution 1: The Real-Time Monitoring Dashboard

Problem: Engineers had no centralised way to monitor factory conditions, forcing them to react to issues rather than prevent them.

Solution: I designed and built a real-time dashboard to give engineers an at-a-glance view of 12 critical data types, from temperature and humidity to machine vibration, using a network of RS485 sensors housed in a custom 3D-printed enclosure.

Process Rigour - Hardware Validation:

To ensure the data was trustworthy, I designed and 3D printed custom jigs in SolidWorks to systematically test and validate the roll, pitch, yaw, and acceleration data from the IMU sensors. This process of rigorous validation was crucial for gaining the engineering team's confidence in the tool.

Solution 2: The Automated QC Tool

Problem: The manual quality control process for composite parts was slow and prone to human error, creating a bottleneck in the production line.

Solution: I developed a computer vision tool that allowed QC technicians to automatically verify part geometry against a defined tolerance.

Key Technical Challenge - Solving Image Distortion:

A major obstacle was that the camera lens warped and distorted the images, making accurate measurement impossible. I researched and implemented a camera calibration routine in Python that mathematically undistorted the images, ensuring the geometric analysis was precise and reliable. This was a critical step that unblocked the entire project.

Impact:

These tools significantly improved the workflows for both the engineering and QC teams. I had the opportunity to present the progress and positive impact of these projects at quarterly review meetings to company stakeholders and external funders, which further developed my technical communication skills.
