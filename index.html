Project 1: EAV Health Monitoring System

A Predictive Dashboard for EV Fleet Managers

The Challenge

Fleet managers of commercial Electric-Assisted Vehicles (EAVs) have no easy way to monitor the long-term health of their vehicle batteries. This leads to unexpected failures, costly downtime, and inefficient maintenance scheduling. They need a tool that can predict battery degradation before it becomes a critical issue.

My Design Process

I started with low-fidelity sketches to explore layouts, focusing on a clear visual hierarchy for a non-technical user. This evolved into digital wireframes to solidify the structure before visual design.

The Proposed Solution

The proposed solution is a clean, intuitive dashboard that provides a predictive "State of Health" (SoH) score for each vehicle in the fleet, allowing managers to quickly identify at-risk vehicles and schedule maintenance proactively.

Key Design Decisions:

  • Prioritised "At-a-Glance" Info: The main SoH score is the largest element, using colour-coding (green, yellow, red) so the manager can assess fleet health in seconds.
  • Action-Oriented List: The vehicle list is sorted by the most critical batteries first, telling the user exactly where to focus their attention.
  • Clear Data Visualisation: The "Health Over Time" graph helps the user spot trends in battery degradation, turning complex data into a simple visual story.

Technical Implementation

The core of the system is a Python script that analyses raw battery data to generate the health score. The logic is clearly commented to be understood by other developers.

# This function takes in raw sensor readings and calculates a weighted SoH score.
def calculate_state_of_health(voltage_readings, temp_readings):
    # Normalize readings
    norm_voltage = normalize(voltage_readings)
    norm_temp = normalize(temp_readings)

    # Apply weighting (example weights)
    voltage_weight = 0.6
    temp_weight = 0.4
    
    health_score = (norm_voltage * voltage_weight) + (norm_temp * temp_weight)
    
    return round(health_score * 100, 2)

Next Steps: Currently, the data analysis module is complete. The next step is to build a functional prototype by connecting this Python backend to a web framework like Flask or Django to serve the dashboard interface.

Project 2: Designing for Engineers at Airborne UK

Improving Workflows with User-Focused Internal Tools

The Challenge

During my year in industry, I was tasked with improving the efficiency of the engineering and quality control teams. I identified two key areas where manual processes were causing delays: factory monitoring and component inspection. As the work was proprietary, all visuals shown are recreated wireframes.

Solution 1: The Monitoring Dashboard

Problem: Engineers had no centralised way to monitor factory conditions, forcing them to react to issues.

Solution: I designed and built a real-time dashboard to give engineers an at-a-glance view of all critical systems.

Impact: This tool moved the team from a reactive to a proactive workflow, saving valuable diagnostic time.

Solution 2: The Computer Vision Tool

Problem: The manual quality control process for composite parts was slow and prone to human error.

Solution: I developed a simple computer vision tool that allowed QC technicians to perform inspections with greater accuracy and speed.

Impact: The tool improved the reliability and efficiency of the quality control process.
